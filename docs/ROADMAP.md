## Фаза 1: Критический путь (Неделя 1)
> Цель: Работающее бронирование с реальными данными

### День 1-2: База данных
- [ ] Развернуть PostgreSQL (Timeweb Cloud или Yandex Cloud)
- [ ] Применить миграции (`migrations/*.sql`)
- [ ] Применить схемы (`database/*.sql`)
- [ ] Настроить переменные окружения (DATABASE_URL)
- [ ] Проверить подключение через `/api/health`

### День 3-4: Бронирование
- [ ] Форма бронирования тура (дата, участники, контакты)
- [ ] API `POST /api/bookings` — сохранение в БД
- [ ] Страница подтверждения бронирования
- [ ] Email уведомление оператору (Resend/Nodemailer)
- [ ] WhatsApp fallback для fishingkam.ru

### День 5: Деплой
- [ ] Настроить secrets в GitHub (TIMEWEB_*)
- [ ] Исправить CI/CD workflow
- [ ] Первый деплой на production
- [ ] Проверить работу на реальном домене

### День 6-7: Тестирование
- [ ] Полный путь туриста: главная → тур → бронь
- [ ] Путь оператора: dashboard → бронирования
- [ ] Исправление багов
- [ ] Оптимизация производительности

**Результат:** Можно бронировать туры fishingkam.ru
