# Lessons Learned & Rules

Этот файл содержит уроки, извлеченные из предыдущих задач, и правила, которые должны соблюдаться всеми агентами.
Дирижёр обязан читать этот файл в начале каждой сессии.

## Правила

1. **Server Components**: Всегда явно указывать `"use server"` в server actions.
2. **Metadata**: Никогда не смешивать `'use client'` и `export const metadata` в одном файле. Использовать паттерн server/client split.
3. **Data Fetching**: Использовать `useSWR` вместо `fetch` в `useEffect`.
4. **Performance**: Использовать `blur(10px)` вместо `blur(20px)` для glassmorphism эффектов.
5. **Accessibility**: Всегда добавлять `htmlFor` и `id` атрибуты для связывания `label` и `input`.
6. **React Keys**: Использовать уникальные идентификаторы (например, `item.id`) вместо индексов массива в качестве ключей.
7. **Render Functions**: Избегать inline функций рендера, выносить их в отдельные именованные компоненты.
